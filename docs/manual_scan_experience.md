- Manual Scan Flow: The user navigates from the Project Hub to the Scan Configuration screen and selects "Manual Scan." They are presented with a checklist of all 11 security stages. As stages are selected, any required input fields (like Target URL or IP) expand inline. The "Run Selected Scans" button remains inactive until all required inputs are provided and validated. Upon execution, the user is redirected to the Live Scan Progress screen, which displays only the selected stages.

- Component Selection UX: Stages are grouped into functional categories to aid navigation: Code Analysis (Sonar stages), Dependency Analysis (NPM/PIP, Dependency Check, Trivy FS), Container Security (Docker stages, Trivy Image), and Active Scanning (Nmap, ZAP). Each stage is represented by a checkbox and a clear, functional label that abstracts away the underlying tool name.

- Input Handling & Validation: Required inputs (e.g., URLs for ZAP, IP addresses for Nmap) appear directly beneath their respective stage labels once selected. Validation occurs in real-time as the user types, checking for valid URI patterns, IPv4/IPv6 addresses, or CIDR ranges. Clear error messages appear if the input format is invalid.

- Invalid Request Handling: The primary execution button is hard-disabled if any required field is empty or fails validation. A global validation summary identifies exactly which stages require attention. If zero stages are selected, the button is disabled with a prompt: "Please select at least one security stage."

- User Feedback Rules: If a selected stage fails to execute (e.g., target host is down), the UI displays a high-level "Execution Failed" status with a simplified reason (e.g., "Target connection timed out") instead of raw tool logs. Every selected stage must report a terminal status: Successful, Failed, or User-Cancelled.

- Forbidden Behaviors: The system must NEVER automatically select or run unselected stages, even if they appear to be dependencies (e.g., NPM Install should not run unless checked). No background auto-discovery or technology fingerprinting is permitted in this mode. Jenkins build numbers and Kali Linux environment details must remain entirely hidden.
