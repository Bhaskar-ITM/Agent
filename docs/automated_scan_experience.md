- Pre-Scan UX: A summary card showing the Project Name, Repository URL, and Target Branch. A prominent "Start Automated Scan" button is accompanied by a brief explanation: "The system will identify your technology stack and run all relevant security checks automatically."
- In-Progress UX: A unified progress bar showing the current functional phase (e.g., "Initializing," "Code Analysis," "Infrastructure Scan," "Finalizing Results"). A live status feed provides human-readable updates such as "Analyzing source code for security patterns..." or "Checking dependencies for known vulnerabilities." No tool names or terminal outputs are shown.
- Post-Scan UX: A high-level summary dashboard displaying the total number of findings categorized by severity (Critical, High, Medium, Low). A "Scan Health" indicator confirms that all applicable checks were completed. A clear call-to-action button "View Detailed Report" leads to the full list of findings.
- Hidden Information: Jenkins build numbers/logs, Kali Linux agent details, individual tool names (Sonar, Trivy, Nmap, ZAP), raw shell commands, and internal pipeline stage names (e.g., "Git Checkout," "Docker Build").
- Skip Communication Rules: Stages that the system determines are inapplicable (e.g., Docker scan for a non-containerized project) are silently omitted from the progress view to maintain a clean interface. In the final report, a "Coverage Summary" section can explicitly list functional areas covered (e.g., "Static Analysis: Completed") without highlighting what was skipped unless it indicates a configuration gap.
